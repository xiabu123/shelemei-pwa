# 射了没 v0.01bata 产品文档

记录日期：2026-06-17
版本名：v0.01bata
版本基线：946c35d Move AI range switch to lower corner
产品形态：React + Vite PWA，Capacitor Android debug 包

## 1. 产品定义

“射了没”是一款面向成年男性的私密身体记录应用。它用很轻的方式记录一次射精事件，并把频率、原因、释放槽、射后状态和诱因整理成可读的趋势。产品语气坦荡、有梗、克制，不做羞辱，不做医学诊断，也不把性体验包装成竞技评分。

v0.01bata 的核心目标是跑通一个能自己玩起来的本地闭环：

1. 打开应用。
2. 首页直接看到距离上次发射、本周节奏和最近记录。
3. 点右下角“冲”进入快捷记录。
4. 在弹层里选择“撸啊撸”或“干一炮”，补充释放槽、状态、标签和备注。
5. 保存后首页、AI射页、记录页立即更新。
6. AI 用本地模拟逻辑给出单次反馈和周期总结。

## 2. 当前版本定位

v0.01bata 是一个本地优先、可安装、可体验的早期 beta。它不是完整商业版，也不是医疗工具。它的价值在于验证交互、视觉调性和数据闭环是否成立。

当前版本重点：

- 快速记录优先。
- 本地数据优先。
- AI 分析先用 mock provider 验证体验。
- Android 先用 debug APK 供个人安装试玩。
- 不做账号、云同步、社交、伴侣实体和医学判断。

## 3. 目标用户

目标用户是愿意观察自己射精行为、冲动触发和射后状态的成年男性。典型场景包括：

- 想知道自己最近冲得是否太频繁。
- 想区分“压力大/无聊导致的撸啊撸”和“真实亲密行为”。
- 想观察射后是放松、困、精神好，还是空虚、焦虑。
- 想把性相关记录未来接入更大的健康、训练、睡眠和自控管理系统。
- 喜欢直接、有梗、但不油腻的个人数据工具。

## 4. 信息架构

应用有三个底部 Tab，只显示图标，不显示文字：

- 首页：状态入口和快捷记录。
- AI射：周期分析、趋势和 AI 总结。
- 记录：历史记录维护。

底部 Tab 固定在手机视口底部。页面主体独立滚动，顶部状态栏留安全区，不再显示大标题栏。这个版本刻意减少“App 标题”存在感，让内容和动作成为第一屏主角。

## 5. 首页

首页没有顶部标题栏，直接从核心状态卡开始。

首页模块：

- 距离上次发射卡：展示“刚刚 / N 小时 / N 天 / 还没发射记录”。
- 7 天节奏条：展示最近 7 天是否有记录，今天用深色强调，有记录根据主要原因上色。
- 指标卡：本周次数、平均释放槽。
- 最近记录：展示最近 2-3 条记录，包含原因、时间和释放槽。
- AI 教练卡：展示最新单次 AI 反馈或轻提示。
- 右下角浮动按钮“冲”：唯一记录入口。
- 首页信息浮窗：点击顶部数据区域可打开产品简介和当前数据说明。

首页交互原则：

- 进入后第一眼看到“距离上次发射”和节奏。
- 右下角“冲”始终可见，避免用户寻找记录入口。
- 不在首页堆复杂数据，复杂分析放到 AI射页。
- 空状态引导用户“右下角点冲，10 秒记完第一条”。

## 6. 快捷记录弹层

点击“冲”后打开快捷记录弹层。弹层默认原因是“撸啊撸”，但用户可以在弹层顶部选择：

- 撸啊撸
- 干一炮

默认 10 秒记录字段：

- 发生时间：默认当前时间，可编辑。
- 发射原因：二选一。
- 释放槽：1-10 格，默认撸啊撸为 6，干一炮为 8。
- 事后感觉：爽 / 一般 / 不太行，单选。
- 事后反应：放松、困、空虚、精神好，多选。
- 保存记录。

展开 30 秒详情字段：

- 为什么会射：睡前无聊、压力大、刷到刺激、想睡觉、氛围到了、对方主动。
- 更多反应：腰酸、后悔、想再来、疲惫、焦虑、满足。
- 随手补一句：自由备注。
- 隐私提示：AI 只看频率、诱因、释放槽和射后状态，不分析伴侣或对象。

保存行为：

- 保存后弹层关闭。
- 本地记录立即写入 localStorage。
- 首页/AI射/记录页从记录实时重新计算。
- 新记录会异步生成单次 mock AI 反馈。
- 如果 3 分钟内重复保存，会提示“刚刚已经记过一发，已保存；重复了可以去记录里删。”，但不会阻塞保存。

## 7. AI射页

AI射页不显示顶部标题栏。页面主体直接展示 AI 分析卡，并在右下角提供固定竖向时间切换。

页面模块：

- AI射分析卡：展示周期总结标题、正文和建议。
- 周期节奏卡：展示 7 天或 30 天内次数、平均间隔。
- 7 天节奏条：继续展示最近 7 天节奏。
- 原因统计：撸啊撸次数、干一炮次数。
- 释放槽趋势：最多展示最近 8 条周期记录的柱状趋势。
- 常见后续反应：Top 标签。
- 右下角 7/30 切换：上方显示“7”，下方显示“30”，视觉只显示数字，按钮 aria-label 保留“7天 / 30天”。

AI射页回答的问题：

- 最近 7/30 天记录了几次？
- 平均释放槽是多少？
- 常见射后反应是什么？
- 主要诱因是什么？
- 下周可以尝试什么轻动作？

## 8. 记录页

记录页不显示顶部标题栏和总数条，直接展示历史记录列表。

记录卡展示：

- 原因。
- 发生时间。
- 释放槽。
- 事后主状态和前两个反应标签。
- 编辑按钮。
- 删除按钮。

编辑行为：

- 点击编辑会把该记录转成 draft，打开同一个快捷记录弹层。
- 用户可修改时间、原因、释放槽、状态、标签和备注。
- 保存后更新时间戳，记录列表重新排序。

删除行为：

- 点击删除弹出浏览器 confirm。
- 确认后移除记录。
- 首页距离、AI射统计和记录列表同步刷新。

## 9. 数据模型

核心类型：

```ts
type EjaculationReason = "撸啊撸" | "干一炮";
type PostState = "爽" | "一般" | "不太行";

interface EjaculationRecord {
  id: string;
  occurredAt: string;
  reason: EjaculationReason;
  pleasureScore: number;
  postState: PostState;
  reactionTags: string[];
  triggerTags: string[];
  note: string;
  aiFeedback?: string;
  createdAt: string;
  updatedAt: string;
}
```

本地存储：

- key：`shelemei.records.v1`
- 存储介质：`localStorage`
- 派生指标不持久化，每次从记录实时计算。
- repository 统一负责保存、更新、删除和排序。

派生指标：

- 最新记录。
- 距离上次发射。
- 今日次数。
- 最近 7 天节奏条。
- 本周次数。
- 平均释放槽。
- 平均间隔天数。
- 原因次数。
- 常见后续反应。
- 常见诱因。

## 10. AI 设计

当前版本使用 `MockAiProvider`。

能力：

- `generateSingleFeedback(record, recentRecords)`：保存新记录后生成一句单次反馈。
- `generatePeriodSummary(records, range)`：AI射页生成 7/30 天总结。

mock 反馈规则：

- 记录里包含“困”时，提示优先休息。
- 诱因包含“睡前无聊”或“刷到刺激”时，提示无屏缓冲。
- 干一炮且释放槽高时，反馈正向但不过度分析。
- 事后状态“不太行”时，提示不要把一次状态当结论。
- 其他情况提示继续观察频率、诱因和射后反应。

未来真实 AI 接入原则：

- 浏览器内不放 OpenAI API Key。
- 通过未来后端 endpoint 调用。
- 只发送必要记录摘要。
- 不上传可识别他人信息。
- 不做医学诊断。

## 11. 隐私与安全

当前版本默认本地保存，不做账号，不做云同步。

隐私边界：

- 不结构化记录伴侣或对象。
- 不鼓励写姓名、联系方式或可识别他人的信息。
- AI 文案不分析伴侣或对象。
- Android Manifest 已关闭 `allowBackup` 和 `fullBackupContent`，降低系统备份暴露风险。
- 当前 Android 包不声明 INTERNET 权限，mock AI 不需要联网。

用户仍需注意：

- 自由备注是本地明文 localStorage 数据。
- debug APK 不适合正式分发。
- 手机丢失、被他人解锁时，本地记录仍可能被看到。

## 12. 视觉与交互规范

视觉关键词：

- 坦荡。
- 有梗。
- 数据感。
- 男性向。
- 克制。
- 不低俗。
- 不黑金电竞。

主色策略：

- 深紫黑：核心强调、今天状态、选中态。
- 淡紫：记录/节奏背景。
- 黄色：干一炮、释放槽填充。
- 淡绿：AI 教练和正向反馈。
- 米白：柔和信息卡。

布局原则：

- 手机优先。
- 内容贴近真实 App，不做营销首页。
- 底部 Tab 固定。
- 首页“冲”按钮固定在右下角。
- AI射页 `7/30` 固定在右下角，并用竖向小控件节省顶部空间。
- 三个主页面都不显示大标题栏，给内容更多首屏空间。

## 13. PWA 与 Android

Android：

- webDir：`dist`
- 当前可安装包：debug APK。
- 当前 package 用途：个人手机试玩，不是正式签名发布。

## 17. vibecoding 亮点记录

这次 vibecoding 最大的亮点是产品调性和工程闭环同时推进，而不是只做一个静态页面。

亮点：

- 从“射了没”这个有风险但有记忆点的名字出发，逐步把表达控制在坦荡、有梗、不低俗的范围内。
- 首页从早期多入口调整为单一“冲”入口，降低记录负担。
- 原因从外部双按钮收敛进弹层，既保留“撸啊撸 / 干一炮”区分，又让首页更干净。
- 7 天节奏条成为产品的核心视觉符号，能快速表达“射了/没射”和不同原因。
- 三个页面陆续去掉顶部标题栏，形成内容优先、手机 App 感更强的结构。
- AI射页把 7/30 切换移到右下角，和记录按钮形成一致的浮动控件语言。
- 通过 browser comments 边看边改，形成了高密度 UI 迭代节奏。
- PWA 和 Android APK 同时跑通，让产品从网页原型进入“手机可装可玩”的阶段。
- Android Manifest 及时收紧备份和网络权限，符合私密记录产品的方向。
- 在 mock AI 阶段就定义了 `AiProvider` 接口，未来接真实后端不会推翻 UI。

## 18. 下次构建注意点

下一次构建前应优先处理这些点：

- 名称拼写：本版本按用户原文记录为 `v0.01bata`，后续如面向外部可考虑统一为 `beta`。
- Android 包：当前是 debug APK，正式内测前应生成签名 release APK/AAB。
- 数据迁移：localStorage key 已确定为 `shelemei.records.v1`，未来改 IndexedDB 或字段结构时要写迁移策略。
- 隐私锁：私密记录产品下一步应优先加应用内锁屏、隐藏标题、快速遮挡或隐私模式。
- 备注风险：自由备注可能记录敏感文本，真实 AI 接入前需要做发送摘要、脱敏和用户确认。
- AI 边界：真实 AI 只能给模式观察和轻建议，不输出诊断或性功能判断。
- 视觉债务：当前图标仍是占位风格，下一步可设计更有品牌感但不擦边的 app icon 和启动页。
- 记录编辑：当前编辑复用弹层，后续可补更清晰的“正在编辑”状态。
- 删除确认：浏览器 confirm 可用但不够原生，后续可做自定义确认弹层。
- 数据页遮挡：右下角 7/30 控件和底部 Tab/内容可能在极小屏上接近，新增模块时要检查遮挡。
- 测试数据：本地已有测试记录时会影响视觉判断，下次截图前应准备一组固定 demo 数据。
- 文档同步：早期 PRD/设计文档仍保留旧的“冲/干双按钮”描述，后续正式文档应以 release 文档为准。

## 19. v0.02 建议路线

优先级建议：

1. 隐私模式：应用内 PIN/生物识别入口、后台模糊、快速隐藏。
2. 数据导出：JSON 导出/导入，避免本地数据丢失。
3. 编辑体验：记录详情页或更强的编辑状态。
4. 真实 AI 后端：只发送摘要，支持单次反馈和周期总结。
5. 统计深化：诱因趋势、时间段分布、连续无射天数。
6. Android release：签名、版本号、安装说明、图标/启动页优化。
